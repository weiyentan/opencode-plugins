## Verdict

**Request changes**

The `PR_FULL_QUERY` has a critical GraphQL syntax error â€” duplicate `commits` field without alias. This must be fixed before the tool can function. The fix requires:

1. Alias the second `commits(last: 1)` to `ciStatus: commits(last: 1)` (or similar)
2. Update the CI status extraction code in `rich.ts` to reference the aliased field name instead of the confusing `pr.commits?.["commits"]` pattern
3. Update the PR test fixtures and add a test case that exercises CI status extraction

The remaining three rich tools (`github.issue.get-full`, `github.issue.search`, `github.repo.get-full`) and `github.query` are well-implemented and clean.
